# 方向一：盈利惊喜预测完整实施方案

## 研究标题（工作标题）
**Do Analyst Forecast Characteristics Predict Earnings Surprises in Asia-Pacific Markets? A Machine Learning Approach with SHAP Interpretability**

---

## 一、核心研究问题

### 1.1 主研究问题
分析师盈利预测与实际业绩之间的偏差（earnings surprise）能否通过**多模态信号**（财报文本 + 结构化财务数据 + 历史预测行为）提前预测？

### 1.2 具体研究假设
- **H1**: 财报MD&A文本的情感特征与未来盈利惊喜方向显著相关
- **H2**: 结构化财务指标（盈利能力、成长性、估值）对盈利惊喜有预测力
- **H3**: 多模态融合模型优于单一信息源模型
- **H4**: 亚太市场的预测模式与美欧市场存在差异（信息效率、披露质量）

### 1.3 核心创新点
1. **数据壁垒**：Capital IQ Pro的Estimates数据（分析师共识预测）+ 实际财务数据，机构级数据质量
2. **跨市场系统性研究**：首个系统比较亚太三市场（新加坡/香港/澳大利亚）分析师预测偏差的ML研究
3. **可解释性**：SHAP全局+局部解释，识别偏差驱动因素（不是黑盒预测）
4. **务实方法论**：多特征集成（财务+分析师行为+情感）+ XGBoost + 跨市场对比，而非复杂多模态架构

## 三、方法论设计

### 3.1 整体架构（简化务实版）
```
特征工程层：
├── 特征组1：结构化财务指标（P/E, ROE, 营收增长等）→ 25-30维
├── 特征组2：分析师行为特征（覆盖度, 预测离散度, 历史准确率）→ 8-10维
├── 特征组3：情感特征（FinBERT预训练提取年报情感分数）→ 3-5维
└── 特征组4：历史surprise序列（滚动统计）→ 8-10维

模型层：
└── XGBoost/LightGBM 集成 → 预测 Surprise_Direction（二分类：正向/负向）

解释层：
└── SHAP全局特征重要性 + 单样本解释 + 跨市场对比分析
```

**关键简化**：
- 不做复杂的多模态融合架构（不需要A40 fine-tune）
- 文本只用预训练FinBERT提取情感分数（不做embedding融合）
- 重心从"方法创新"转向"金融发现+跨市场比较"

### 3.2 文本处理流程（简化版）

**只做情感分数提取，不做fine-tuning**
- 使用**预训练FinBERT**提取年报/财报的情感分数（positive/negative/neutral概率）
- 不提取768维embedding，只用3个情感概率作为特征
- 工具：`transformers` library，`ProsusAI/finbert`模型
- 处理对象：年报（如果季报MD&A不可获取）

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch

tokenizer = AutoTokenizer.from_pretrained("ProsusAI/finbert")
model = AutoModelForSequenceClassification.from_pretrained("ProsusAI/finbert")

def extract_sentiment_scores(text):
    inputs = tokenizer(text, return_tensors="pt", truncation=True, max_length=512)
    with torch.no_grad():
        outputs = model(**inputs)
    probs = torch.nn.functional.softmax(outputs.logits, dim=-1)
    # 返回 [positive_prob, negative_prob, neutral_prob]
    return probs[0].numpy()
```

**为什么简化**：
- 避免复杂的多模态融合架构（3-4个月时间不够）
- 重心从"方法创新"转向"金融发现+跨市场比较"
- 情感分数已被证明对金融预测有效，reviewer容易接受

### 3.3 结构化特征工程

#### 特征分组
| 特征类别 | 具体特征 | 计算方式 |
|---------|---------|---------|
| **盈利能力** | ROE, ROA, Net Margin, Operating Margin | 直接从Capital IQ |
| **成长性** | Revenue Growth (YoY, QoQ), EPS Growth | (当期-上期)/上期 |
| **估值** | P/E, P/B, EV/EBITDA | 直接从Capital IQ |
| **财务健康** | Debt/Equity, Current Ratio, Interest Coverage | 直接或计算 |
| **历史surprise** | Prior Surprise (t-1, t-2, t-3, t-4) | 滚动窗口 |
| **分析师行为** | Number of Analysts, Forecast Dispersion | 直接从Estimates |
| **市场反应** | Cumulative Return (过去30天) | 从价格数据计算 |

#### 特征预处理
- 缺失值：用行业中位数填充
- 异常值：Winsorize（1%-99%分位数）
- 标准化：StandardScaler（Z-score归一化）

## 四、实验设计

### 4.1 主实验
**RQ1**: 多模态模型是否优于单模态？
- 对比：XGBoost(Text+Structured) vs XGBoost(Structured-only) vs FinBERT(Text-only)
- 指标：F1-score, Accuracy

**RQ2**: 哪些特征最重要？
- SHAP分析：全局特征重要性 + 单样本解释
- 分组分析：文本特征 vs 财务特征 vs 历史surprise

**RQ3**: 亚太市场是否有独特模式？
- 分市场训练：新加坡 vs 香港 vs 澳大利亚
- 跨市场泛化：在A市场训练，在B市场测试

## 七、风险管理与备选方案

### 7.1 潜在风险点

#### 风险1：Capital IQ文本数据覆盖不足
**表现**：MD&A文本只有年报，没有季报；或者亚太公司文本质量差
**应对**：
- Plan A：改用earnings announcement press release（Capital IQ新闻数据库有）
- Plan B：只用结构化特征 + 历史surprise，放弃文本部分
- Plan C：切换到方向四（破产预测），不依赖文本

#### 风险2：多模态融合效果不显著
**表现**：XGBoost(Text+Structured) 的F1-score只比Structured-only高1-2个百分点
**应对**：
- 调整研究角度：不强调"多模态融合"，改为"分析师预测偏差的驱动因素分析"
- 用SHAP分析深挖哪些特征最重要，转向解释性研究
- 仍然可以投Pacific-Basin或Finance Research Letters

#### 风险3：样本量不足
**表现**：筛选后只有200-300家公司，数据点 < 3000
**应对**：
- 扩大地域范围：加入韩国、台湾、印度
- 降低筛选条件：分析师覆盖 ≥ 2（而非3）
- 改为case study：深度分析新加坡单一市场

#### 风险4：审稿人质疑"数据窥探"
**表现**：审稿人认为用未来信息（如t期的MD&A预测t期的surprise）
**应对**：
- 严格时间对齐：用t-1期的MD&A和t-1期的财务数据预测t期的surprise
- 在论文中明确说明时间窗口设计
- 提供时间线图（Figure 1）
